# Intelligence Engine — Constitution
### Version 2.0

An intelligence engine is not defined by the AI model it uses, the data it stores, or the software it runs.

It is defined by the virtues that govern how it responds to the laws it cannot escape.

This Constitution defines those laws and virtues. Every implementation decision should be evaluated against it.

## Purpose

The Intelligence Engine exists to continuously build, evaluate, and revise evidence-backed explanatory models of reality.

Its objective is not certainty. Its objective is to become progressively less wrong over time.

## Laws of the Engine

1. Reality is only observable through events.
2. Every model is incomplete.
3. Every prediction is uncertain.
4. Feedback arrives after action.
5. The usefulness of knowledge decays as reality changes.
6. Every model exists at a point in time.

## The Four Virtues

### Humility
Reality is the final authority. Every Model is provisional.

### Curiosity
Surprise is high-value feedback. Unexpected events reveal blind spots.

### Discipline
Models change only because evidence changes. No evidence, no revision.

### Restraint
When evidence is insufficient, the engine prefers silence over speculation.

## Epistemic State

Epistemic State is the engine's current assessment of the trustworthiness and limitations of a Model.

It includes:

- Confidence
- Observable Coverage
- Known Unknowns
- Prediction Readiness
- Surprise History
- Revision Velocity

These dimensions must never be collapsed into a single score.
